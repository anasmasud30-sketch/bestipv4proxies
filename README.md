# BestIPv4Proxies — Static Proxy Comparison Site

A fast, fully **static** proxy comparison website for **bestipv4proxies.com**. Built with plain
HTML, CSS and a small amount of vanilla JavaScript — **no PHP, no Node.js, no database, no build step.**
Just upload the folder to any static host and it works.

The site is a comparison/affiliate hub that ranks proxy providers, keeping our own service
**Cheapest Proxies** in the #1 position and listing competitors below it.

---

## Design

- **Dark theme** with a "network / terminal" aesthetic: near-black canvas, a faint grid texture,
  teal → cyan → violet gradient accents, monospace micro-labels and gold rating stars.
- Animated four-part **score bars** on each review (Value / Speed / Coverage / Support).
- Glassy cards, sticky blurred header, dropdown navigation, FAQ accordion, scroll-reveal
  animations and a back-to-top button.
- Fully responsive (desktop → mobile) and accessible (skip link, ARIA labels, reduced-motion support).

## Deploy

It's static, so any of these work with zero configuration:

1. **Drag-and-drop** the folder into Netlify, Cloudflare Pages, Vercel, GitHub Pages, etc.
2. **Upload via FTP** to any web host's public root (`public_html` / `www`).
3. **Preview locally** — just open `index.html` in a browser, or run a tiny server:
   ```bash
   # Python 3
   python -m http.server 8080
   # then visit http://localhost:8080
   ```

## Customising

- **Colours / theme:** edit the design tokens in the `:root` block at the top of `assets/css/styles.css`.
- **Providers / rankings / prices:** edit the comparison table and the `<article class="review">`
  blocks in `index.html`. Each review's score bars use inline `style="--v:NN%"` to set bar width.
- **Email / brand / domain:** search-and-replace `info@affordableproxyhub.com`, `BestIPv4Proxies`
  and `bestipv4proxies.com`.
- Add a real raster `og-cover.png` (1200×630) under `assets/img/` and point the `og:image` tags at it
  for richer social sharing (currently they reference the SVG logo).

## Programmatic pages (`_build/generate.py`)

The site includes **275 HTML pages**. Most are generated by `_build/generate.py`, a local
Python authoring script (run `python _build/generate.py`). It is NOT part of the deployed
site — the output is plain static HTML. It produces, each on a unique entity with unique
title/meta/H1 and 20 content sections:

- **60 country pages** — `proxies-in-<country>.html`
- **40 website pages** — `proxies-for-<site>.html` (Instagram, Amazon, sneakers, Google…)
- **30 use-case pages** — `<use-case>-proxies.html` (web scraping, SEO, ads…)
- **16 proxy-type pages** — `<type>-proxies.html` (residential, datacenter, mobile, IPv4…)
- **30 comparison pages** — `<a>-vs-<b>.html`
- **24 provider reviews** — `<provider>-review.html`
- **50 blog posts** — `blog-<topic>.html`
- **6 hub pages** — `proxies-by-country/website/use-case.html`, `proxy-types.html`,
  `provider-reviews.html`, `proxy-comparisons.html` (linked from the nav "Proxies" menu)

The script also rebuilds `sitemap.xml` and rewrites the shared nav/footer across the
hand-written pages. Internal linking is dense (hubs → children, children → siblings + hubs +
homepage). **Note on SEO:** these are programmatic pages with shared structure and varied,
entity-specific copy. Hand-edit your highest-priority pages (top countries/platforms, your
own review) with original detail to maximise rankings and avoid thin/duplicate-content risk.
To change wording at scale, edit the content pools in `_build/generate.py` and re-run.
